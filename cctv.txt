라즈베리파이 cctv

라즈베리파이에서 cctv로 차량이 인식되는 화면을 관리자만 볼 수 있도록 만들어 두었어! 그래서 이제 웹 화면에 영상의 도메인만 추가하면 웹페이지 관리자   
   페이지에서 확인할 수 있도록 만들고 싶어! 관리자 페이지에서 현재 충전소 cctv화면을 확인할 수 있는 공간만 만들어주면 이후 라즈베리파이에서 도메인을    
   가져와 cctv를 직접확인할 수 있도록 하고 싶은데                                                                                                       
                                                                                                                                                        
   이 부분에 대해서 아주 자세하게 분석해서 세부적으로 체계적으로 설계해서 꼼꼼하게 계획해줘!

✦ 라즈베리파이 기반의 AI 비전 CCTV 스트리밍을 관리자 페이지에 연동하는 것은 스마트 관제 시스템(VIUM)의 '실시간 모니터링 가치'를 극대화할 수 있는 아주
  훌륭한 기획입니다.

  시니어 아키텍트 관점에서, 라즈베리파이의 스트리밍을 안정적으로 웹에 올리기 위해 프론트엔드 UI를 어떻게 구성하고, 추후 도메인만 갈아끼울 수 있도록
  아키텍처를 어떻게 설계해야 하는지 [CCTV 연동 마스터 플랜]을 꼼꼼하게 브리핑해 드립니다.

  ---

  1. 기술적 사전 분석 (Technical Analysis)

  라즈베리파이에서 송출하는 실시간 영상은 보통 MJPEG(Motion JPEG) 스트리밍 방식이나 WebRTC, 혹은 HLS 방식을 사용합니다. 
  가장 보편적인 MJPEG 방식(Flask 등을 이용한 스트리밍)을 가정했을 때, 웹에서는 무거운 동영상 플레이어 없이 단순한 <img> 태그나 <iframe> 태그만으로
  영상을 렌더링할 수 있습니다.

   * 현재 상태: 라즈베리파이에서 스트리밍 서버(예: 포트 5000번 등)가 구동 중.
   * 목표 상태: 관리자가 Admin 계정으로 로그인 시 대시보드에서 해당 영상을 실시간 시청.
   * 핵심 고려사항: 
       1. 라즈베리파이가 꺼져있거나 통신이 끊겼을 때 화면이 흉하게 깨지지 않도록 에러 핸들링(Fallback UI) 필수.
       2. 여러 대의 CCTV가 생길 것을 대비해 충전소 선택형(Dropdown) UI로 확장성 확보.

  ---

  2. 세부 UI/UX 아키텍처 설계 (Architecture Design)

  대상 파일: vium-web/src/components/admin/AdminDashboard.tsx

  현재 관리자 대시보드 상단(또는 별도의 '관제 센터' 탭)에 [실시간 CCTV 모니터링 위젯]을 배치합니다.

  📺 [CCTV 모니터링 위젯 UI 구성안]
   1. 헤더 영역 (Header)
       * 빨간색 깜빡이는 원(🔴 REC / LIVE) 배치로 실시간 느낌 부여.
       * 관제 대상 충전소 선택 드롭다운 (예: 양주 신도8차 아파트 1호기 ▼)
   2. 영상 출력 영역 (Player Container)
       * aspect-video (16:9 비율)의 검은색 배경 박스.
       * 정상 연결 시: <img> 태그의 src 속성에 라즈베리파이 도메인 주소 바인딩.
       * 연결 대기/실패 시: "CCTV 신호를 대기 중입니다..." 또는 "연결 오프라인" 이라는 Placeholder(대체 화면) 노출.
   3. 오버레이 정보 (Overlay)
       * 영상 좌측 상단: 현재 시각 (예: 2026-05-19 14:30:22)
       * 영상 우측 상단: 현재 AI 인식 상태 (예: 차량 인식됨 (98%) - 추후 API와 연동 가능)

  3. 단계별 구현 계획 (Execution Plan)

  나중에 라즈베리파이 도메인만 딱 꽂으면 완성되도록 미리 그릇을 만들어두는 작업 순서입니다.

  Step 1: 환경 변수 및 상태(State) 준비
  코드 어딘가에 주소를 하드코딩하지 않고, 상단에 변수로 빼두어 관리하기 쉽게 만듭니다.

   2 // 💡 추후 이 문자열 안에 라즈베리파이의 실제 도메인 주소를 넣으시면 됩니다.
   3 const [cctvUrl, setCctvUrl] = useState<string>("http://[라즈베리파이_IP]:[포트]/video_feed");

  영상이 끊겼을 때 엑스박스(X)가 뜨지 않도록 방어하는 로직입니다.

    1 // 이미지 로딩 실패 시 호출되는 함수
    2 const handleCctvError = () => {
    3     setIsCctvOnline(false);
    4 };
    5
    6 // ... JSX 렌더링 부분 ...
    7 <div className="relative w-full aspect-video bg-gray-900 rounded-2xl overflow-hidden border-4 border-gray-800 shadow-2xl">
    8     {isCctvOnline ? (
    9         <img 
   10             src={cctvUrl}

   16         <div className="absolute inset-0 flex flex-col items-center justify-center text-gray-500">
   17             <VideoOff size={48} className="mb-4 opacity-50" />

   20             {/* 재연결 시도 버튼 */}
   21             <button onClick={() => setIsCctvOnline(true)} className="mt-4 px-4 py-2 bg-gray-800 rounded-lg hover:bg-gray-700 text-white text-xs">

   27     {/* LIVE 뱃지 (영상 위에 겹침) */}
   28     <div className="absolute top-4 left-4 bg-black/50 backdrop-blur px-3 py-1 rounded-full flex items-center gap-2">
   29         <span className={`w-2 h-2 rounded-full ${isCctvOnline ? 'bg-red-500 animate-pulse' : 'bg-gray-500'}`}></span>
   30         <span className="text-white text-xs font-black tracking-widest">LIVE CCTV</span>
   31     </div>
   32 </div>

  Step 3: 보안 및 HTTPS 통신 제약 대비 (가장 중요한 부분 ⚠️)
   * Mixed Content 경고 대비: 만약 우리 VIUM 웹사이트를 나중에 HTTPS(보안 연결)로 배포하게 되면, 브라우저 정책상 HTTP(보안 없는)로 송출되는
     라즈베리파이의 영상을 차단해버릴 수 있습니다.
   * 해결책: 라즈베리파이에서 영상을 송출할 때도 ngrok을 켜서 https:// 로 시작하는 주소를 발급받은 뒤 그 주소를 cctvUrl에 넣는 것이 가장 안전하고 확실한
     방법입니다.

 요청하신 대로 아주 신중하고 꼼꼼하게 관리자 대시보드에 실시간 CCTV 관제 위젯 작업을 완료했습니다! 

  기존의 단순했던 탭 UI를 개편하여 제일 앞단에 'CCTV 실시간 관제' 탭을 신설했으며, 다음과 같은 디테일들을 완벽하게 구현했습니다.

  🌟 주요 업데이트 내용
   1. 16:9 딥다크 시네마틱 UI (Aspect-Video): 실제 관제 센터의 느낌을 주도록 어두운 배경과 얇은 테두리를 적용하여 영상의 몰입감을 높였습니다.
   2. 실시간 LIVE 뱃지: 위젯 좌측 상단에 빨간색으로 깜빡이는(Pulse) 애니메이션 뱃지를 달아 실시간 스트리밍 중임을 직관적으로 보여줍니다.
   3. 완벽한 에러 방어선 (Fallback UI): 라즈베리파이가 꺼져있거나, 아직 도메인을 입력하지 않아 영상이 깨질 경우를 대비해 'CCTV 연결 오프라인' 이라는
      세련된 대체 화면이 뜨도록 방어 로직(onError)을 짰습니다.
   4. 영상 재연결 버튼: 통신이 일시적으로 끊겼을 때 새로고침 없이 영상만 다시 불러올 수 있는 '재연결 시도' 버튼을 추가했습니다.
